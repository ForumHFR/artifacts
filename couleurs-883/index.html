4 · Ce qui ne change pas / points d'attention

Invariants. Primaire rouge (les 2 accents, défaut + « Rouge RF1 »), sémantique erreur, couleurs de drapeaux, clamp de lisibilité des [color] d'auteurs. Contrastes on/container vérifiés AA (T1 : 8,6:1 clair / 7,3:1 sombre ; T2 : 9:1 / 6,4:1). AMOLED : fond noir pur conservé, mais les containers restent légèrement relevés (tout à #000000 détruirait la hiérarchie).
À verrouiller à l'implémentation. Position sur le Dynamic Color (remplace tout / surfaces seules / désactivé) pour que la palette soit réellement canonique ; vérification des surfaces à élévation réelle (tonal elevation), des états pressed/selected/disabled/ripple/scrim ; contour neutre systématique sur les chips de drapeaux (un blanc/rouge/jaune peut disparaître selon le fond). Un réglage multi-teintes (3 × 2 accents × 3 modes = 18 rendus) est DÉCONSEILLÉ — matrice de test disproportionnée pour une préférence esthétique.

5 · Recommandation

A + T2 (choix aussi de la revue Sol) : le rouge conserve l'identité, l'ardoise élimine franchement le grief « jaune » — là où A + T1 risque d'être perçu comme « le même moutarde mieux maquillé ». A + T1 reste l'alternative conservatrice. B = la parité RF1 assumée, à garder éventuellement pour plus tard comme mode « RF1 gris » expérimental — pas comme réglage jour 1.

Maquettes indicatives (rendu web ≈, pas du Compose). Implémentation : Color.kt seul + captures avant/après par écran clé ; aucun composant à toucher (tout passe par les rôles M3).
